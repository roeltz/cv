HTML5 + CSS + JS, LESS, jQuery, RequireJS, PHP 5.6, Apache 2.4, NodeJS 0.10 y +4.5, Express 3 y 4, Socket.IO, Grunt, ActionScript 3, MySQL 5.5, MongoDB 2.4, Twitter REST API, Facebook Graph API, C++ (NodeJS Addons y Win32)

Logros Notables

  • Una plataforma de Señalización Digital (SD): Inicialmente la compañía licenciaba software comercial de SD de terceros, y nuestro sistema sólo se limitaba a alimentar, desde un módulo de gestión de contenidos, información actualizable en tiempo real (texto e imágenes) a los spots dinámicos (e.g. calendarios de eventos o menús digitales). Eventualmente logré convencer a mis superiores de desarrollar, por nuestra cuenta, el software de reproducción, y el resto es historia: el sistema creció hasta convertirse en una plataforma, con un gestor de contenidos que permite monitorear y manipular equipos conectados, programar listas de reproducción (incluyendo hilo musical), gestionar colas de espera, e integrar información del clima, redes sociales y muchas otras fuentes. El software cliente de reproducción es multiplataforma (Windows, Linux x64 y ARM), permite la sincronización para desplegar videos entre múltiples pantallas e incluye un modo hub en el que un equipo sirve de proxy para acelerar las descargas, entre otras cosas simpáticas.
  • Sistemas periféricos: La plataforma ha crecido para incluir herramientas para apoyar los flujos de trabajos internos de creación de contenidos y soporte técnico, así como para atender necesidades muy específicas de los clientes. Entre ellos: una librería JS para la creación de documentos HTML dinámicos de forma declarativa y sencilla, orientada a diseñadores gráficos; un sistema de actualización y renderización automática de proyectos de After Effects (AutoRender) para spots publicitarios que contienen data dinámica; y un clúster autobalanceado de transcodificación de videos.
  • El sitio web de la compañía, imvinet.com: desarrollé todo el concepto visual, redacté y traduje el copy al inglés, y rehice la plantilla del blog corporativo en Blogger para encajar con el estilo del sitio. No, no es responsive, lo sé.

Proyectos personales

Estos proyectos son desarrollados en mi tiempo libre con el objetivo de aprender, explorar nuevos conceptos y formar una base de código para asumir proyectos con mayor productividad. Muchos de éstos proyectos están alojados en mi perfil de GitHub (github.com/roeltz).

  • Kakoto, un foro online, con tópicos de conversación, posts y réplicas anidadas, y un chat con notas de voz, sonidos y GIFs. Está construido como una PWA, genera notificaciones push y guarda borradores de posts localmente.
  • wlessc, una utilidad de línea de comandos para observar y compilar archivos LESS. Es una herramienta sencilla, que integra PostCSS con Autoprefixer y CleanCSS para prefijar y minificar el CSS resultante. Parece ser mi único proyecto popular, con más de 7K descargas en npm.
  • tp, una biblioteca JS para plantillas basadas en DOM. Fue desarrollado por la necesidad de tener un motor de plantillas extremadamente ligero, eficiente y extensible para presentar contenido dinámico en un Raspberry Pi modelo B.
  • Pipa, un framework MVC y ORM para PHP 5.4. Ha sido mi caballo de trabajo desde el 2013, y surgió de mi inconformidad con las alternativas —que considero sólo buscan copiar Rails o Spring. Tiene componentes de enrutamiento y vistas bastante flexibles, pensados para crear APIs REST. El módulo de ORM es poderosísimo, con soporte para bases de datos relacionales (MySQL, PostgreSQL, SQL Server) y de documentos (MongoDB) y puede resolver relaciones transitivas entre entidades en las consultas. La única debilidad de éste proyecto es que no tiene documentación, aunque eso no ha impedido que sea usado por algunos colegas.
  • mt, una utilidad de línea de comandos para convertir modelos de MySQL Workbench en scripts de creación de tablas para PostgreSQL, SQL Server y clases de entidad para Pipa, extensible para entradas y salidas adicionales, gracias a su representación abstracta de esquemas.
